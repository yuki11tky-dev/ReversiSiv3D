# ReversiSiv3D

### 概要

本ポートフォリオはタイトルの通りゲームの「リバーシ」になります。実際に業務の中でc++を使用する経験に恵まれず業務経験は短いですが、個人でゲームエンジンでのc++の扱い方やネイティブのc++の言語仕様については日頃勉強をしております。今後c++を使用した開発を経験したいと考えることと業務での使用経験は浅いですがアプリケーション開発ができることのアピールをするために、ポートフォリオはc++での開発を選択しました。開発物はゲームですが操作感ではなく、設計や実装内容を中心にご確認いただければと思います。実際に遊べるように実装しております。実行環境がお手元にある場合はぜひ実行して動作もご確認いただければと思います。

画像やサウンド周りの実装に関しては扱いやすくクラスでラップされているOpenSiv3Dというライブラリを使用して実装しています。OpenSiv3Dについてはこれまで使用経験はなく関数定義を確認したり、チュートリアルやドキュメントを確認しながら扱い方を学びながら実装しました。

### ディレクトリ構成
```
ReversiSiv3D
 └─ ReversiSiv3D
     ├─ App
     ├─ Asset
     └─ Reversi
```

 - Appフォルダ
   - ビルド生成物とアセット群がこのフォルダに配置
   - ビルド生成物の他にOpenSiv3D関連のdll等も配置
 - Assetフォルダ
   - ゲーム中で使用する画像ファイルやサウンドファイルを配置
   - ビルド後イベントでサブフォルダ以下がAppフォルダにコピーされる
     - これはOpenSiv3Dの既定ディレクトリがAppになっているため
 - Reversiフォルダ
   - プログラムのヘッダーファイルやソースファイルが配置

### 実行時の注意点

ゲーム実行中にはBGMやSEが流れますのでゲーム実行する場合は音量設定にご注意ください。

ゲームを実行すると`.\ReversiSiv3D\ReversiSiv3D\App\config\`ディレクトリにゲーム設定の**config.json**ファイルが出力されます。config.jsonからサウンドの調節が可能です。詳細は以下になります。
 - Debug StartLevel
   - デバッグ実行時に開始できる画面を変更できます
   - 画面切り替えは以下の通りです
     - 0 : 開始画面(BootLevel)
     - 1 : タイトル画面(TitleLevel)
     - 2 : ゲーム画面(BattleLevel)
     - 3 : リザルト画面(ResultLevel)
 - Sound Volume
   - ゲーム中のサウンドボリュームの調整ができます※BGMとSEの両方の統一設定になります
     - 0.0(消音) ～ 1.0(最大) の間で設定してください
 - WindowSize
   - ゲームウィンドウのサイズ設定が変更可能です
     - 変更可能なサイズは以下の通りです
       - 1920 * 1080 (推奨)
       - 1200 * 800
       - 800 * 600

3. 各画面上での責務分離
4. 描画システム

詳細は設計ドキュメントをご覧ください。

### 最後に

このように階層を分けた設計にて実装することで将来の機能拡張を実施しやくすること、また各クラスの責務を明確に分離しているため問題発生箇所を追跡しやすく保守性にも優れたものになると考えています。また今回のポートフォリオでは存分には表現できなかったのですが、ファイル入出力や処理中の例外処理や排他制御に対しても意識して実装することができます。例外が発生する箇所に関しては例えばtry-catchを使用する場合はどの範囲で捕捉するべきかなど例外安全性を考慮した実装、排他制御の場合はロックする範囲を限定しパフォーマンスに影響が出ないように実装することができます。現在の業務にてメインで使用しているC#に関しても機能拡張や修正対応時には最適な対応方法を検討し、影響範囲を考慮した実装を心掛けています。
